Syntax

# at_cascade.cascade_root_node(
    all_node_database       = None,
    root_node_database      = None,
    fit_goal_set            = None,
    no_ode_fit              = False,
# )

all_node_database

is a python string specifying the location of the all_node_db relative to the current working directory. This argument can’t be None.

root_node_database

is a python string specifying the location of the dismod_at root_node_database.

fit_goal_set

This is a set with elements of type int (str) specifying the node_id (node_name) for each element of the fit_goal_set . This argument can’t be None.

no_ode_fit

if True ( False ) the Do A No Ode Fit For One Node routine will (will not) be used to modify the parent value priors.

root_fit_database

This directory is

result_dir/root_node_name/dismod.db

see result_dir and root_node_name. If no_ode_fit is True ( False ) the priors in the root_node_database are modified (are not modified) before the root node is fit. Upon return, this is a fit_node_database with the extra properties listed below:

Output dismod.db

There is a dismod.db output file for every row in the job_table. Upon return, the results for this fit are in dismod.db in the corresponding directory; see Get Database Directory Corresponding To a Fit. The .csv files can be created using the dismod_at db2csv command. The dismod_at function plot_rate_fit and plot_data_fit can be used to crate the corresponding plots.

fit_var

The fit_var table correspond to the posterior mean for the model variables for this job; i.e., this fit_node_id and split_reference_id.

sample

The sample table contains the corresponding samples from the posterior distribution for the model variables for this job.

c_predict_fit_var

The c_predict_fit_var table contains the predict table corresponding to the predict fit_var command using the avgint table in the root_node_database except that values in the node_id column has been replaced by this fit_node_id. Note that the predict_id column name was changed to c_predict_fit_var_id (which is not the same as var_id).

c_predict_sample

The c_predict_sample table contains the predict table corresponding to the predict sample command using the avgint table in the root_node_database except that values in the node_id column has been replaced by this fit_node_id. Note that the predict_id column name was changed to c_predict_sample_id (which is not the same as sample_id).

log

The log table contains a summary of the operations preformed on dismod.db between it’s input and output state.
