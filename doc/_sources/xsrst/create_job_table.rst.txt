.. _create_job_table.purpose:

Purpose
*******
This routine returns a list of shift (node_id, split_reference_id) pairs
with a corresponding fit (node_id, split_reference_id) pair.
Each shift pairs requires the corresponding fit pair to have completed before
it can be run.

.. _create_job_table.start_split_reference_id:

start_split_reference_id
************************
This, together with *start_node_id*
corresponds to a completed fit that we are starting from.
Only jobs that depend on the start jobs completion will be included in
the job table.

fit_node_id
===========
This is the node_id for the :ref:`glossary.fit_node` for this *this_job_id*.

.. meta::
   :keywords: split_reference_id

.. index:: split_reference_id

.. _create_job_table.job_table.split_reference_id:

split_reference_id
==================
If the split_reference table is empty, this is ``None``.
Otherwise it is the :ref:`split_reference_table.split_reference_id`
for this *this_job_id*; i.e. the splitting covariate has this reference
value.

.. meta::

.. index:: parent_job_id

.. _create_job_table.job_table.parent_job_id:

parent_job_id
=============
This is the job_id corresponding to the parent job.
The parent job (and only the parent job)
must have completed before this job can be run.
