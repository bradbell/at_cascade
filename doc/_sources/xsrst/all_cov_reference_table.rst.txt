All Node Covariate Reference Table
##################################
.. contents::
   :local:

This table specifies the reference value for every covariate
and node in the :ref:`glossary.root_node_database`.
It has the following columns:

.. meta::
   :keywords: all_cov_reference_id

.. index:: all_cov_reference_id

.. _all_cov_reference_table.all_cov_reference_id:

all_cov_reference_id
********************
is the :ref:`all_node_db.primary_key` for this table.

.. meta::
   :keywords: node_id

.. index:: node_id

.. _all_cov_reference_table.node_id:

node_id
*******
This column has type ``integer`` and specifies
the node for this reference
by the primary key in the root_node_database node table.

.. meta::
   :keywords: covariate_id

.. index:: covariate_id

.. _all_cov_reference_table.covariate_id:

covariate_id
************
This column has type ``integer`` and specifies
the covariate, for this reference,
by the primary key in the root_node_database covariate table.

.. meta::
   :keywords: split_reference_id

.. index:: split_reference_id

.. _all_cov_reference_table.split_reference_id:

split_reference_id
******************
This column has type ``integer`` and it specifies the
index of a value in
:ref:`all_option_table.split_list.split_reference_list`.
If there is (is not) a split_list in the option table,
the values in this column must not be (must be) null.

.. meta::
   :keywords: reference

.. index:: reference

.. _all_cov_reference_table.reference:

reference
*********
This column has type ``real`` and it specifies the reference value
for this covariate and node.
For each node_id in the :ref:`glossary.fit_node_set`,
each covariate_id,
and each split_reference,
there must be one and only one reference value.
Note that when the covariate_id corresponds to the split_covariate,
the reference and the split_reference must have the same value.
